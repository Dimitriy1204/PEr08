# RAG-ассистент (Yandex Embeddings + GigaChat)

Консолидированный RAG-ассистент на базе Yandex Embeddings API и GigaChat-2 от Сбера, с векторным хранилищем ChromaDB и предварительной фильтрацией по источнику.

## Архитектура

```
📦 корень проекта
├── app.py              # Точка входа (интерактивный / демо-режим)
├── embeddings.py       # Yandex Embeddings + ChromaDB (cosine-метрика)
├── rag.py              # RAG-ассистент с GigaChat-2
├── search_config.py    # Централизованное управление фильтрацией 🔥
├── cache.py            # Кеш ответов (JSON)
├── evaluate_ragas.py   # Оценка качества через RAGAS
├── env.example         # Шаблон конфигурации
├── .env                # Файл с API-ключами (в gitignore)
├── requirements.txt    # Зависимости
├── .gitignore          # Игнорируемые файлы
└── DATA/               # База знаний (единственная)
    ├── PEr01_common_info.txt
    └── PEr01_FAQ.txt
```

## Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Настройка .env

Скопируйте `env.example` в `.env` и заполните ключи:

```bash
cp env.example .env
```

Минимально необходимые переменные:

| Переменная | Описание |
|---|---|
| `YANDEX_API_KEY` | API-ключ Яндекс Облака |
| `YANDEX_FOLDER_ID` | ID каталога в Яндекс Облаке |
| `GIGACHAT_CREDENTIALS` | Учётные данные GigaChat |

### 3. Запуск

```bash
python app.py
```

При первом запуске:
1. Создаётся коллекция ChromaDB с cosine-метрикой
2. Документы из папки `/DATA` загружаются в векторную БД
3. Запускается интерактивный режим

### 4. (Опционально) Удалить старую БД

Если у вас была предыдущая версия с L2-метрикой:

```bash
rmdir /S /Q chroma_db   # Windows
rm -rf chroma_db         # Linux/Mac
```

## Команды интерактивного режима

| Команда | Описание |
|---|---|
| **любой вопрос** | Задать вопрос ассистенту |
| `filter <имя>` | Установить ручной фильтр (например: `filter PEr01_FAQ.txt`) |
| `filter_off` | Отключить все фильтры — поиск по всем документам |
| `auto_filter on` | GigaChat сам определяет, в каком файле искать ответ |
| `auto_filter off` | Отключить авто-определение темы |
| `stats` | Статистика системы |
| `cache` | Количество записей в кеше |
| `clear_cache` | Очистить кеш |
| `exit / quit / q` | Выход |

### Приоритет фильтров

1. **Ручной** `filter <имя>` — самый высокий, сбрасывает авто-фильтр
2. **Авто** `auto_filter on` — сбрасывает ручной
3. **Без фильтра** `filter_off` или по умолчанию — поиск по всем документам

## Фильтрация по источнику

### Как это работает

1. **Предварительный фильтр ChromaDB** — where-условие на уровне индекса, ДО семантического поиска. Это быстрее, чем фильтрация результатов после поиска.
2. **Авто-определение темы** — GigaChat-2 получает запрос и список доступных источников с их описаниями (`SOURCE_DESCRIPTIONS` в `rag.py`), определяет, к какому файлу относится вопрос, и устанавливает фильтр.

### Добавление новых описаний источников

В `rag.py` добавить запись в `SOURCE_DESCRIPTIONS`:

```python
SOURCE_DESCRIPTIONS = {
    "PEr01_common_info.txt": "Общая информация об учебном центре...",
    "PEr01_FAQ.txt": "Вопросы и ответы по ЧПУ Fanuc...",
    "новый_файл.txt": "Краткое описание содержания"
}
```

## Замер времени

Каждый запрос замеряется через `time.monotonic`. Для быстрых операций (кеш) — в миллисекундах (`123.4 мс`), для медленных (RAG-генерация) — в секундах (`2.345 сек`).

## Структура базы знаний

Папка `/DATA` — единственная база знаний проекта. Любой `.txt` файл в ней автоматически загружается в ChromaDB при первом запуске:

- `PEr01_common_info.txt` — информация об учебном центре «Точка отсчета» и «Завод Стройтехника»
- `PEr01_FAQ.txt` — вопросы и ответы по системе ЧПУ Fanuc

Для добавления нового источника достаточно положить `.txt` файл в `/DATA`, очистить коллекцию и перезапустить приложение.

## Технологии

- **Yandex Embeddings API** — векторные представления текста
- **GigaChat-2** (Сбер) — генерация ответов (через `langchain-gigachat`)
- **ChromaDB** — векторная БД с cosine-метрикой
- **Python 3.11+**

## Оценка качества (RAGAS)

```bash
python evaluate_ragas.py
```

Запускает оценку по метрикам Faithfulness и Context Precision на тестовых вопросах из папки `/DATA`.
